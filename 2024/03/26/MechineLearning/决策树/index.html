决策树

前面我们已经学习了线性回归、逻辑回归和多级分类，本文我们再来学习一种监督型的机器学习模型——决策树模型。

一、 决策树模型

我们现在手上有10张卡通动物的图片，根据图片可以总结出每个动物的耳朵形状，脸型和是否有胡子，也可以判断出这个动物是否是猫。

  2. 如果是尖尖耳朵，那就再判断脸型，如果是圆脸就得出结论是猫。如果不是圆脸就得出结论不是猫。
  3. 如果是软软耳朵，那就再判断胡子，如果有胡子就得出结论是猫，如果没有胡子就得出结论不是猫。

这样，我们从树的根节点开始，一直走到叶子节点得出结论，就能够判断。这就是决策树模型

决策树模型能够通过简单的判断决策来进行预测。

当然，如果我们改变了判断条件的顺序，对应的决策树也会相应发生改变。

这些都是决策树。

二、 构建决策树

知道了什么是决策树模型，下一步就应该开始构建了。如何构建出来一个预测效果好的决策树，我们有两个问题:

  1. 从上面我们可以看到，同一个问题可以构建出来的决策树不止一个。如何选择每一个节点的特征值？
  2. 决策树在什么时候应该停止决策，也就是什么时候应该定义叶子节点。

我们带着这两个问题继续往下走。

1、 计算纯度、熵和信息增益

每当我们从一个特征值开始决策的时候，数据都会被分为两部分。

比如，我们选择耳朵的形状就会得到

左边有5个小动物，其中4只是猫；右边也有五个小动物，其中1只是猫。

我们定义P1 为 数据里是猫的占比， 也可以说是数据里猫的纯度。

  • 在左边，P1 = 4/5 = 80%
  • 在右边，P1 = 1/5 = 20%

而如果我们以是否有胡子进行决策

那么会得到

  • 左边P1 = 4/7 = 57%
  • 右边P1 = 1/3 = 33%

纯度有什么用呢？当纯度越高或越低的时候， 说明这个特征越能很好地区别。比如P1=100%的时候就可以直接作出判断是猫， 而P1=0%的时候也可以直接作出判断不是猫。

所以，我们在选择特征值的时候，要尽量优先选择那些决策后纯度较高或较低的特征值。

为了更好地描述，我们引入了熵的概念用来衡量数据的纯度， 熵的函数如下:

h(x)=−xlog2x−(1−x)log2(1−x)h(x) = -xlog_2x - (1-x)log_2(1-x) h(x)=−xlog2​x−(1−x)log2​(1−x)

熵的函数图像如下:

从图像我们可以发现，纯度越高和越低, 熵就越小。在纯度=0.5时，熵最大，H(0,5)=1

这样我们就可以用熵来描述一个特征值决策后的效果。

因为决策后数据被分为两部分，所以我们用加权平均来表示，例如在用耳朵区分的例子中

10个数据被一分为二，左边一共有5个数据，其中4个是猫;右边一共5个数据, 其中1个是猫。

那加权平均熵就是:

510H(45)+510H(15)=0.729\frac{5}{10}H(\frac{4}{5}) + \frac{5}{10}H(\frac{1}{5})=0.729 105​H(54​)+105​H(51​)=0.729

而如果选择的是脸型

那么加权平均熵就是

710H(47)+310H(13)=0.9651\frac{7}{10}H(\frac{4}{7}) + \frac{3}{10}H(\frac{1}{3}) =0.9651 107​H(74​)+103​H(31​)=0.9651

能比较出“耳朵形状”的加权平均熵比“脸型”的胶圈平均熵要小， 所以我们应该优先选择“耳朵形状
”作为节点的特征值。

接着，注意在根节点的中，十只小动物里有五只猫，所以我们也可以计算出根节点的熵:

P1=5/10H(5/10)=1.0P_1=5/10\\ H(5/10) = 1.0 P1​=5/10H(5/10)=1.0

接着我们用根节点的熵 减去 分支的加权平均熵， 比如选择的是“耳朵形状”，那么有

H(5/10)−（510H(45)+510H(15)）=0.28H(5/10) - （\frac{5}{10}H(\frac{4}{5}) + \frac{5}{10}H(\frac{1}{5})） = 0.28 H(5/10)−（105​H(54​)+105​H(51​)）=0.28

这里表示经过特征分类后，熵减少了多少，我们称之为信息增益(Information gain)。

总结一下

  • 用P1表示数据的纯度，也就是目标类型在总数据集里的占比。

  • 用H(P1)表示一个节点的熵， 用于衡量数据的纯度。公示为

    H(p)=−plog2p−(1−p)log2(1−p)H(p) = -plog_2p - (1-p)log_2(1-p) H(p)=−plog2​p−(1−p)log2​(1−p)

  • 用根节点的熵减去子节点的加权平均熵，得到信息增益。

能够计算这些之后， 我们提出的第一个问题就可以解决了。

我们应该计算所有特征值的信息增益，然后选择信息增益最大的特征值作为当前节点。

2、 构建决策树的步骤

知道了如何通过计算纯度、熵和信息增益之后，我们就知道如何选择特征值作为节点了。而构造决策树也就是每一次选择特征值的过程， 所以我们可以用递归的方式来构建，步骤：

  1. 从剩余的特征值中分别信息增益
  2. 检查是否需要停止决策， 如果需要则停止并构造叶子节点
  3. 如果不需要，则选择信息增益最大的特征值作为该节点的特征值
  4. 数据分类后对子节点继续进行该操作。

3、何时停止分裂

在构造决策树的时候，何时停止分裂很重要，他是跳出递归的出口，也能防止模型过度拟合。

我们的例子里只有3个特征值，而实际问题中特征值可能会很多，并且就算分到最后也得不到数据纯度为100%或0%的数据， 那我们就需要决定何时停止分裂，也就是何时添加叶子节点。

  1. 在数据纯度为100%或0%时。这是容易理解的，在训练过程中经过决策后已经能够完成分出目标种类了，那么就可以直接给出结论了。
  2. 在决策树的深度超过设定的最大深度时。决策树的深度是指从根节点到叶子节点的距离。你可以设置最大深度为特征值的数量， 不过这样的话往往会导致决策树过于复杂从而导致过度拟合。所以需要选择一个合适的最大深度。其中一种方法就是通过交叉验证来选择。
  3. 当数据增益已经小到小于阈值的时候。这时候说明再继续分裂也不会有改善了。
  4. 当分裂后的子节点中样本数量太少，达不到最小样本阈值数。 当样本数过小也无法对模型继续进行改善。

三、回归树

决策树是用来做分类的，它通过对特征值的判断，最后预测出一个结果， 这个结果是离散的。类似于逻辑回归和多级分类。 如果我们想要用树来预测连续的值，像线性回归一样， 那我们可以使用回归树。

回归树和决策树的结构上并没有什么区别。我们还是拿上面的例子来说明

四、为什么要用树？

有了线性回归和逻辑回归之后，为什么决策树和回归树的优势在哪里呢？

我们可以发现，决策树和回归树的特点在于，他们善于从离散的特征值中学习规律，做出预测。而离散的值，倘若换成0，1，2，3带入到线性回归里或逻辑回归里， 模型可能会认为他们之间有大小关系，导致预测不准。

所以，如果你的特征值大多是离散的，可分类的， 那么选择决策树或回归树的效果会更好一些。
