从上面的图中，可以很清晰的看到决策树的结构和判断逻辑。

首先会判断花蕊的长度，如果长度小于等于2.45cm， 那么就判定为setosa鸢尾花。

如果长度大于2.45cm， 那么再看宽度。如果宽度小于等于1.75，则判定为versicolor鸢尾花，否则判定为virginica鸢尾花。

图中的几个参数含义如下:

  • samples: 当前节点的实例数量

  • value: 当前节点各个种类的数量

  • class: 当前节点判断的种类， 如果非白色的是叶子节点，就代表当前节点给出的预测。

  • gini: 基尼系数, 用于衡量不纯度. 他的计算公式为:

    Gi=1−∑k=1npi,k2 G_i = 1-\sum_{k=1}^n p_{i,k}^2 Gi​=1−k=1∑n​pi,k2​

    其中p_i,k表示第i个节点中训练实例之间的k类示例的比率

4. CART训练算法

Scikit-Learn是使用CART(Classification and Regression Tree)算法来构造决策树的。
他的原理是使用单个特征k的某个阈值t_k， 将数据分为两类。

比如花蕊长度(k) <= 2.45(t_k)

那么如何知道哪个特征k的哪个阈值t_k，能把数据分为两个子集呢？并且是要分类后尽量纯度最高

CART算法采用下面的公式计算成本:

J(k,tk)=mlmGleft+mrmGrightJ(k, t_k) = \frac{m_l}{m}G_{left} + \frac{m_r}{m}G_{right} J(k,tk​)=mml​​Gleft​+mmr​​Gright​

其中G就是Gini函数， 计算不纯度。 这个成本函数越高，说明两个子集的纯度越低。

不理解的也可以看一下用熵来计算 决策树计算熵

分为两个子集后， 除非到达了设置的最大深度max_depth， 或者某一子集的纯度以及很高了， 那么就会停止运行。

所以CART算法是一个贪心算法， 他能得到一个比较好的解， 但是没办法得到最好的解。而且对数据的变化也非常敏感， 添加少量数据就可能使得最后的决策树模型完全不一样。

5. 计算复杂度

决策树的预测特别快，因为只需要遍历节点， 所以预测大约会经过O(log_2(m))个节点。

训练时因为会计算每个特征分后的纯度， 所以训练的复杂度为O(n x m log_2(m))

6.正则化超参数

如果不对决策树进行控制， 那么训练出来的决策树可能会过拟合。这时候就需要正则化。

我们可以通过限制max_depth 决策树的最大深度来减少过拟合的可能。

DecisionTreeClassifier还有一个其他的参数

  • min_samples_split 分裂前节点必须有的最小样本数
  • min_samples_leaf 叶子节点必须要有的最小样本数
  • min_weight_fraction_leaf 叶子节点必须要有的 加权实例占比样本数
  • max_leaf_nodes 最大叶子节点的数量
  • max_features 分裂每个节点评估的最大特征数量

通过增大min或减小max都能使模型正则化。
