在这个例子中，第一行表示所有的非5图片， 其中有53892个图片被正确识别为非5(真负类)， 687张图片被错误识别为5(假正类)
第二行表示所有5的图片， 其中有1891个图片被错误识别成了非5(假负类)， 3530张图片被正确识别为5(真正类)
一个完美的分类器，混淆矩阵应该是这样的:
[[54579 0]
[ 0 5421]]

分类器的精度

精度=TPTP+FP精度=\frac{TP}{TP+FP} 精度=TP+FPTP​

其中 TF是真正类的数量， FP是假正类的数量，在本例中， 精度=3530 / (3530+687) = 0.83

精度只在乎正类， 会忽略其他信息。 还有一个指标是召回率

召回率=TPTP+FN召回率=\frac{TP}{TP+FN} 召回率=TP+FNTP​

FN是假负类的数量
在本例中， 召回率 = 3530 / (3530+1891) = 0.65

0.8370879772350012
0.6511713705958311

上面的结果表示， 这个检测器，当他说一张图片是5时， 只有83%的准确率， 并且只检测出了65%的数字5.

我们可以把精度和召唤率合并成一个单一指标F1

F1=21精度+1召回率=2∗精度∗召回率精度+召回率=TPTP+FN+FP2F_1 = \frac{2}{\frac{1}{精度} + \frac{1}{召回率}} = 2*\frac{精度*召回率}{精度+召回率} = \frac{TP}{TP+\frac{FN+FP}{2}} F1​=精度1​+召回率1​2​=2∗精度+召回率精度∗召回率​=TP+2FN+FP​TP​

0.7325171197343847

不过，具体的场景下对精度和召回率的要求不同。
比如在青少年模式下的视频推送，需要训练一个分类器来检测是否适合青少年观看。这时候你宁愿拦截掉了一些好视频， 也要保证放出来的全是适合青少年观看的。即需要高精度， 可以接受低回归率。

而在识别小偷的系统上，宁愿放弃一些精度， 也要保证能识别到所有的小偷。 即低精度， 高回归率。
当然你没办法同时提高精度和回归率。

3.3.5 ROC曲线

对于二元分类器， 还可以用ROC曲线来评估。ROC曲线，叫受试者工作特征曲线
ROC曲线绘制的是真正类率 和 假正类率 之间的关系。
有点绕，慢慢解释。
真正类率 TPR， 是召回的另一个名称。 指被正确分为正类的比例。

TPR=TPTP+FNTPR = \frac{TP}{TP+FN} TPR=TP+FNTP​

假正类率 FPR，指被错误分为正类的比例。 在本例子中， 也就是不是5却被识别成了5的比例。

FPR=FPFP+TNFPR = \frac{FP}{FP+TN} FPR=FP+TNFP​

假正类率也等于 1-特异度， 特异度就是 被正确识别为负类的比例

特异度=TNFP+TN特异度 = \frac{TN}{FP+TN} 特异度=FP+TNTN​

​

​

从图中我们可以看出来， 召回率越高， 会导致假正率也变大。

有一种评估分类器的方法是计算ROC曲线下的面积（AUC）。 完美的分类器ROC AUC = 1， 而纯随机分类器的ROC AUC = 0.5

3.4 用多个二元分类实现多类分类器

我们最终是要检测一张图片是0-9的某一个数字，而并非检测是否是其中一个数字。 我们这里尝试用多个二元分类器来实现。

一种方法是对于n类问题， 我们可以实现n个二元分类器， 分别判断每一个类的概率。在本例子中即0-检测器， 1-检测器， 2-检测器…
哪一个检测器检给的正类分数最高， 就将图片归为哪个类。 这是一种一对多的策略OvR(one versus all)

另一种方法是，为每一对数字训练一个分类器。即一个用于区别0和1，一个用于区别0和2… 这样我们就需要N*(N-1)/2个分类器。
当对一个图片进行分类的时候，看哪一个数字胜出最多，就归为哪个类。 这是一种一对一的策略OvO。
OvO的优点在于，每个分类器只需要用到部分训练集(即需要分类的两个类别)对其训练即可。

使用sklearn的svc可以自动选择OvO或OvR
